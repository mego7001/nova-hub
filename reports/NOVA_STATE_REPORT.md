# NOVA STATE REPORT
Date: 2026-02-06

## صورة عامة
Nova Hub دلوقتي شغال كنظام مساعد هندسي حيّ، بيتعامل مع البشر بطريقة هادية ومفهومة. الحوار طبيعي، والعقلية واضحة: يفهم قبل ما يتحرّك، ويستأذن قبل أي تنفيذ، ويحمي المستخدم والبيئة قبل كل حاجة.

## Nova تقدر تعمل إيه دلوقتي
- محادثة طبيعية (عربي مصري أساسي، وإنجليزي ثانوي) مع ردود قصيرة وواضحة.
- فهم النية والشك والمخاطر، مع رفض مهذب عند اللزوم.
- اقتراح حلول مدعومة بأدلة من الملفات داخل مساحة العمل.
- تشغيل مهام طويلة بشكل آمن، مع نقاط توقف واضحة قبل أي تنفيذ حساس.
- فحص أمني شامل لمساحة العمل مع تقارير واضحة.
- رسم ثنائي الأبعاد من وصف لغوي، مع معاينة ثم تطبيق، وتصدير DXF.
- إدخال صوتي وإخراج صوتي بطريقة آمنة (Offline أولًا، مع تدهور آمن لو المحركات مش موجودة).
- سجل زمني للتغييرات والقرارات والنتائج داخل المشروع.

## نوع المشاكل اللي Nova تقدر تحلها
- مراجعة مشروع برمجي: تنظيم، اقتراحات، تلخيص، وشرح المخاطر.
- إعداد خطط إصلاح خطوة بخطوة قبل التنفيذ.
- فحص أمني داخلي للمجلدات داخل مساحة العمل.
- تحويل وصف هندسي إلى رسم 2D قابل للتصدير.
- تتبّع ما حدث داخل المشروع بشكل واضح وشفاف.

## Nova فاهمة إيه على المستوى المفاهيمي
- الفرق بين الكلام العادي والتنفيذ الفعلي.
- فكرة المخاطر وإدارة القرارات على مراحل.
- مفهوم الثقة وحدود مساحة العمل.
- الفصل بين الوضع الحواري، وضع المشروع، ووضع التنفيذ.
- مفهوم التحسين الذاتي الآمن (اقتراح وخطة قبل التطبيق).

## كيف Nova بتفكّر
- بتقيّم نية المستخدم أولًا: هل المطلوب نصيحة أم تنفيذ؟
- لو الطلب فيه مخاطرة أو تضارب مع الهدف: تعترض بهدوء، تشرح السبب، وتقترح بديل.
- عند الإصرار: تحذير تدريجي من 3 مراحل، والمستخدم يقدر يكمّل بإذن صريح.
- لو حصل فشل: تدخل وضع الاستعادة، تصلح أولًا، ثم تذكير هادي إن ده كان متوقّع، ثم تكمل.

## وضع الأمان والحماية
- تنفيذ محصور داخل مساحة العمل فقط.
- لا تنفيذ من المحادثة الحرة بدون تأكيد صريح.
- الذكاء أونلاين مطفّي افتراضيًا، ولا يُستخدم إلا عند الحاجة وبموافقة واضحة.
- الأسرار لا تُطبع ولا تُسجّل، ويتم تنقيحها تلقائيًا في السجلات والتقارير.
- فحص أمني يكتشف: حدود المساحة، الروابط المشبوهة، مفاتيح حساسة، ملفات خطرة، وإعدادات تنفيذ غير آمنة.
- عند وجود مشكلة حرجة: يتم تعطيل تفعيل الوضع الأونلاين للمشروع لحين إعادة الفحص.

## حدود معروفة
- الفهم الهندسي المعقّد في الرسم قد يحتاج دعم أونلاين إذا الوصف كان بعيد عن القواعد البسيطة.
- خصائص الصوت تعتمد على توافر محركات محلية؛ بديل آمن متوفر بدون تعطل.
- سجل الأحداث يعطي أثر واضح، لكنه لا يمنع العبث خارج النظام إن حدث تعديل يدوي للملفات.

## كيف Nova بتحمي المستخدم
- تمنع التنفيذ التلقائي.
- تُصعّد التحذير بشكل منطقي ومنظم.
- توضح السبب والمخاطر قبل أي خطوة.
- تُعيد السيطرة للمستخدم في كل قرار حساس.

## الخطوة القادمة (تطوّر طبيعي)
- دعم رسم 3D مبدئي ومعاينة تفاعلية.
- طبقة CAD حقيقية للحسابات الهندسية الدقيقة.
- محاكاة بسيطة للنتائج قبل التطبيق.
- تقارير مخاطر أكثر ذكاءً مع سياق زمني أعمق.

## Appendix (Internal)
- Audit spine uses append-only records with hash chaining for tamper-evidence.
- Offline-first policy remains the default for parsing and voice.

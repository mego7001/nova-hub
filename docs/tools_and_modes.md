# Tools and Modes

## Task Modes

Modes are defined in `core/ux/task_modes.py` and availability is computed from loaded tools.

Current mode IDs:

- `general`
- `build_software`
- `gen_3d_step`
- `gen_2d_dxf`

Unavailable modes are hidden in UI selectors.

## Routing Format

Mode routing is generated by `core/ux/mode_routing.py`.

Message wrapper format:

```text
[[NOVA_MODE mode=<mode>; ui=<ui>; scope=<scope>]]
<original message>
```

Properties:

- Reversible (`parse_mode_wrapped_message`).
- Human-readable in logs.
- Does not bypass existing send boundaries.

## Tools Catalog

Tools menu is built from `core/ux/tools_catalog.py` using registry + policy + mode metadata:

- `Curated`: compact, high-frequency actions.
- `Advanced`: full registry grouped by tool group/category.

Source of truth for mode tags and curated mappings:

- `core/ux/tools_registry.py`

Badge semantics:

- `available`: callable directly under policy.
- `approval`: allowed but approval-gated.
- `unavailable`: denied by current policy/profile.

## Upload Policy Coupling

Tool/mode UX works with upload policy in `core/ux/upload_policy.py`:

- validates file types
- enforces quotas by context (`general` vs `project`)
- returns actionable rejection reasons for UI

## UI Integration Points

- HUD: `ui/hud_qml/controller.py`, `ui/hud_qml/qml/components/CommandBar.qml`
- Chat: `ui/chat/app.py`
- Quick Panel: `ui/quick_panel/app.py`
- Compatibility launcher alias: `run_whatsapp.py`
